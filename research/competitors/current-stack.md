# Current Tool Stack — What Exists and Where It Fails

> Mapping of tools researchers use today, based on user complaints and official docs.

| Tool | Best for | Covers well | Fails at |
|------|----------|-------------|----------|
| Zotero | Reference management + PDFs | Citations, PDF annotation, notes | Discovery, tag cleanup, synthesis |
| Mendeley | References + collaboration | Groups, annotation | Stability, migration, lock-in |
| EndNote | Writing + publishing (institutional) | CWYW integration | Discovery, cost, modern UX |
| Obsidian | Personal knowledge base | Linked notes, flexibility | No native citations, plugin-dependent |
| Paperpile | Google Docs + PDF cloud | Browser capture, Drive sync | LaTeX workflows, paid only |
| ResearchRabbit | Visual discovery | Maps, recommendations, tracking | Synthesis, annotation management |
| Connected Papers | Explore from seed paper | Quick field graph | Library management, free limits |
| Litmaps | Map + monitor topics | Visualizations + alerts | Synthesis linked to notes |
| Semantic Scholar | Search + alerts | General discovery | End-to-end workflow |

## Key gap

No tool covers the full workflow. Discovery tools don't connect to reading. Reading tools don't connect to synthesis. Synthesis tools don't exist.
